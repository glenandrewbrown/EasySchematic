1gTouch & safe-area technical spec
Hit targets ≥44px — port handles keep the 9–11px glyph but get an invisible expanded touch radius (::after 44×44 centered); all chrome buttons ≥40px at Tier C, rows ≥44px.
Gestures — pinch = zoom (zoomOnPinch), two-finger drag = pan, one-finger drag on node = move, long-press 500ms = context menu, double-tap = fit selection. Marquee select only via Select tool.
Safe areas — viewport-fit=cover; tab bar + sheets pad with env(safe-area-inset-*); FAB cluster offsets above the tab bar.
Input hygiene — touch-action: manipulation on all controls (kills 300ms/double-tap zoom); -webkit-tap-highlight-color: transparent; hover-only affordances get press equivalents (tooltips → long-press label).
Keyboard-optional — every ⌘K command reachable via visible UI at Tier C; ⌘K itself works with external keyboards on tablet.
Acceptance — 390×844 / 768×1024 / 1024×768 / 1440×900: place device → connect (tap-tap) → edit in sheet → read BOM. No horizontal body scroll, no gate, no sub-44px target.

5fMotion, focus & interaction states
Transitions — colour/bg/border/shadow .14s ease; transform .16s cubic-bezier(.23,1,.32,1). Press: buttons scale(.96), clickable rows scale(.98).
Popovers — espop 160ms from opacity:0 scale(.96) with anchor-correct transform-origin. Occasional menus stagger children 25ms (cap 150ms). ⌘K opens with NO animation (keyboard-invoked, high frequency).
Canvas motion — flow dash (see 2d) 3s / 1.7s selected; connect preview esdash; swatch glow 2s (see 2b); wireless TX rings 2.4s ease-out ×2 staggered; re-fit glide 180ms.
Reduced motion — every animation above gated by prefers-reduced-motion; staggered items must land at opacity:1.
Focus — :focus-visible 2px --color-accent ring, 2px offset, everywhere. Scrollbars: 9px rounded thumb on --ui-border-strong.
Persistence — all view prefs under one easyschematic.ui.v1 key (theme, tier prefs, sheet detent, unit, nodeView, toggles…).
